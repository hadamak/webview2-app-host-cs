CORS プロキシテスト (CDP版)

ホストの proxyOrigins に登録された外部オリジンへの fetch() が透過的に動作することを確認します。
CDP (Chrome DevTools Protocol) により、GET 以外のメソッドやリクエストボディ、カスタムヘッダもサポートされています。
← index.html へ戻る

① GET — JSON データ取得

fetch('https://jsonplaceholder.typicode.com/todos/1')
—

② GET — クエリパラメータ付き

fetch('https://httpbin.org/get?foo=bar&baz=123')
—

③ POST — ボディ転送 (JSON)

fetch('https://httpbin.org/post', { method: 'POST', body: '{...}' })
—

④ PUT — データ更新

fetch('https://httpbin.org/put', { method: 'PUT', body: '{...}' })
—

⑤ ヘッダ — 送受信テスト

X-Custom-Request を送信し、
X-Custom-Response を受信できるか
—

⑥ エラー — 404 Not Found

fetch('https://httpbin.org/status/404')
—

⑦ POST — フォーム形式

application/x-www-form-urlencoded
—
