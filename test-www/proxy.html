CORS プロキシテスト

ホストの proxyOrigins に登録された外部オリジンへの fetch() が、コンテンツ側の URL 変更なしに動作することを確認します。
ブラウザで開いた場合はネットワーク接続があれば通常の CORS リクエストが試みられ、CORS エラーになります（プロキシは機能しません）。
← index.html へ戻る

① GET — JSON データ取得

fetch('https://jsonplaceholder.typicode.com/todos/1')
—

② GET — クエリパラメータ付き

fetch('https://httpbin.org/get?foo=bar&baz=123')
—

③ POST — ボディ転送テスト

fetch('https://httpbin.org/post', { method: 'POST', body: '{...}' }) → CDP プロキシによりボディも転送されます
—

④ 未登録オリジン（ブロック確認）

fetch('https://example.com/') → proxyOrigins 未登録
