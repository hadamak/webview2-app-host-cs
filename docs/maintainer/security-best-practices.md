# Security Best Practices

WebView2AppHost は強力なネイティブ連携機能を提供するため、悪意ある構成の変更からアプリケーションを保護することが極めて重要です。

## 1. app.conf.json の保護と配布形態

`app.conf.json` は、ホストがどの実行ファイルを、どのような引数で起動するかを決定する「特権的な設定ファイル」です。

### 脅威: 構成ハイジャック
攻撃者が `www/app.conf.json` を勝手に配置し、本来安全なサイドカー設定を `cmd.exe` 等に書き換えることで、PC 上で任意のコマンドを実行（RCE）されるリスクがあります。

### 対策: パッケージ化（連結ZIP）による保護モード
本番環境での配布時は、必ず **EXE 末尾に ZIP を連結（パッケージ化）** してください。

連結 ZIP が検出されると、ホストは自動的に **「セキュリティ保護モード」** で動作します：
- **構成の固定化**: `app.conf.json` は連結 ZIP 内のものだけが採用され、外部の `www/app.conf.json` による上書きは物理的に遮断されます。
- **優先順位の強制**: 同名のファイルが ZIP と `www/` フォルダの両方に存在する場合、常に ZIP 内のファイルが優先されます。

### メディアファイル（www/ フォルダ）との併用
セキュリティ保護モード中であっても、動画や音声などの巨大なアセットは引き続き `www/` フォルダに置いて運用することが可能です。

| コンテンツ種類 | 推奨配置場所 | 理由 |
| :--- | :--- | :--- |
| **app.conf.json** | **連結 ZIP** | **必須。** 外部からの改ざんを防止するため。 |
| **プログラム (HTML/JS/CSS)** | **連結 ZIP** | 実行ロジックの完全性を担保するため。 |
| **巨大なメディア (MP4/WAV等)** | **www/ フォルダ** | メモリ節約と高速なシーク（Range Request）のため。 |

## 2. サイドカーの最小権限原則

### CLI モードの引数制限
`cli` モードで `{args}` を使用すると、JavaScript から任意の引数を注入できます。これを `cmd.exe` や `powershell.exe` と組み合わせると、実質的なシェルを提供することになります。

- **対策**: 可能な限り、OS 標準のシェルを介さず、開発者が作成した専用の実行ファイルやスクリプトを直接指定してください。
- **入力のバリデーション**: ユーザー入力をサイドカーに渡す場合は、サイドカー側のスクリプト（Python や Node.js）内で厳密なバリデーションを行ってください。

## 3. 安全なビルド構成（SecureRelease）

高度なセキュリティが要求されるオフライン専用アプリケーションや、サードパーティ製プラグインを一切許可しない用途では、`SecureRelease` 構成でのビルドを検討してください。

- `SecureRelease` では、`SidecarConnector`, `McpConnector`, `PipeConnector`, `CdpProxyHandler` のコードがコンパイル時に完全に削除されます。
- ホスト側にそれらを動かすロジックが存在しないため、たとえ構成が改ざんされても攻撃が成立しません。

## 4. インストール先の保護

アプリケーションは、一般ユーザーが書き込み権限を持たない `C:\Program Files` 以下などにインストールすることを推奨します。これにより、EXE 自体の書き換えや、パッケージ化されていない場合の `www/` フォルダの作成を OS レベルで防止できます。
